制定日: 2026年6月19日

Ringed Seal Studio（以下「当方」といいます）は、モバイルアプリケーション「トクミル」、本ウェブサイトおよびこれらに付随するサービス（以下「本サービス」といいます）における利用者（以下「ユーザー」といいます）の情報の取り扱いについて、本プライバシーポリシー（以下「本ポリシー」といいます）を定めます。

## 1. 基本方針

本サービスは、価格比較に必要な範囲へ情報の取得を限定します。

- 初回利用時はFirebaseの匿名認証を使用します
- アカウント保存を選んだ場合だけ、メールアドレスをFirebase Authenticationへ登録します
- バーコードと通常の文字認識は、原則として端末上で処理します
- 位置情報は任意機能でのみ使用します
- 公開価格情報にはFirebaseのユーザーIDを含めません
- カメラ画像や生の購入金額を行動分析へ送信しません

## 2. 取得する情報と利用目的

| 情報 | 取得する場面 | 利用目的 | 主な保存先 |
|---|---|---|---|
| FirebaseユーザーID | 初回起動・ログイン | ユーザーの識別、データのアクセス制御 | Firebase Authentication / Firestore |
| メールアドレス | ユーザーがアカウント保存を選んだ場合 | アカウント保存、別端末での復元、パスワード再設定 | Firebase Authentication |
| カメラ映像 | バーコード・文字認識 | 商品コードや棚札文字の端末内認識 | 端末メモリ上の一時処理 |
| 撮影画像 | パッケージ認識、棚札価格のクラウド認識を実行した場合 | 商品候補または価格の解析 | Google Gemini APIでの一時処理 |
| JANコード、商品名、検索条件 | 商品検索時 | 商品の特定、EC価格検索、検索結果の一時キャッシュ | 端末、Cloud Functions、Firestore |
| 閲覧履歴、マイリスト、節約記録 | ユーザーが各機能を利用した場合 | 履歴表示、再検索、買い物管理、節約レポート | 端末内。同期・共有機能を使う場合はFirestore |
| 家族グループ情報 | 家族共有機能を使う場合 | 招待、メンバー管理、共有記録 | Firestore |
| 位置情報 | 近隣店舗・地域価格機能を許可した場合 | 店舗候補の特定、近隣価格の表示 | 精密座標は端末メモリ上の一時処理。約100m単位へ丸めた位置をCloud Functionsへ送信 |
| 店舗・商品・価格情報 | 店頭価格を確定し、登録条件を満たした場合 | 地域価格の表示、価格傾向の集計 | Firestore |
| アプリ利用状況 | 分析への同意が有効な場合 | 利用状況の把握、機能改善、不具合箇所の特定 | PostHog Cloud EU |
| クラッシュ・診断情報 | アプリで障害が発生した場合 | 障害調査、品質改善 | Firebase Crashlytics |
| ウェブアクセス情報 | 本ウェブサイトを閲覧した場合 | GitHub Pagesによる配信・セキュリティ維持 | GitHubのシステムログ |

パスワードはFirebase Authenticationが処理し、当方が平文で取得・保存することはありません。

## 3. カメラ画像の取り扱い

バーコード検出と通常の文字認識は、Google ML Kitを利用して端末上で処理します。

以下の機能をユーザーが実行した場合は、対象画像をCloud Functions経由でGoogle Gemini APIへ送信します。

- パッケージ画像からの商品候補の認識
- 棚札画像からの価格認識

当方は、これらの画像をFirestoreやFirebase Storageへ恒久保存することを目的としていません。通信中および解析事業者側での取り扱いは、Googleの利用規約・プライバシーポリシー・データ処理条件に従います。

## 4. 位置情報と地域価格

位置情報は、ユーザーが権限を許可し、近隣店舗または地域価格に関する機能を使用した場合だけ取得します。

1. 精密な座標を端末上で取得します
2. 店舗候補との距離計算など、必要な一時処理に使用します
3. Cloud Functionsへ送る前に、緯度・経度を約100m単位へ丸めます
4. 丸めた位置をGoogle Places APIへ送信し、周辺店舗候補を取得する場合があります

地域価格として保存する情報には、商品識別情報、店舗、価格、観測日、約100m単位の位置、認識の根拠などが含まれます。公開価格レコードにはFirebaseユーザーIDを含めません。

位置情報を許可しなくても、バーコード検索やEC価格比較などの基本機能を利用できます。

## 5. 行動分析

本サービスは、機能改善のためPostHog Cloud EUを利用します。

- アプリ内の同意状態に基づいて送信します
- 設定画面から停止できます
- FirebaseユーザーIDを匿名識別子として使用する場合があります
- セッション録画は使用しません
- カメラ画像、メールアドレス、精密位置情報、商品名、検索語、購入金額の生値は自作分析イベントへ含めません
- 金額を分析する場合は範囲へ丸めた区分値を使用します

アプリの起動や端末・OS・アプリバージョンなど、SDKが付加する技術情報が送信される場合があります。

## 6. クラッシュ・診断情報

Firebase Crashlyticsに、次の情報が送信される場合があります。

- クラッシュやエラーの内容
- スタックトレース
- アプリバージョン
- OS・端末に関する技術情報
- FirebaseユーザーID

これらは障害の再現・修正に使用します。

## 8. 保存期間

| 情報 | おおよその保存期間 |
|---|---|
| 端末内の閲覧履歴・マイリスト・節約記録 | アプリ内削除、アカウント削除、アプリデータ削除またはアンインストールまで |
| Firebase上のユーザー・アカウント情報 | アカウント削除まで |
| 家族共有の節約記録 | 原則2年 |
| 公開価格情報 | 原則90日 |
| EC検索結果キャッシュ | 空結果は約3分、キーワード検索は約6時間、JAN検索は約24時間 |
| JANから特定した商品情報 | 原則30日 |
| 店舗候補キャッシュ | 店舗ありの場合は原則60日。空結果は短期間 |
| PostHog・Crashlyticsの記録 | 各サービスの設定および保持方針に従う期間 |
| GitHub Pagesの配信ログ | GitHubの保持方針に従う期間 |

FirestoreのTTL削除は期限後すぐに実行されず、一定の遅延が生じる場合があります。アプリ側では期限切れデータを利用しないよう判定します。

法令上の義務、セキュリティ、不正利用防止、紛争対応に必要な場合は、必要な範囲でこれより長く保持することがあります。

## 9. 第三者提供

当方は、次の場合を除き、ユーザーの情報を本人の同意なく第三者へ販売・提供しません。

1. 本ポリシーに記載した外部サービスを利用して本サービスを提供する場合
2. 法令に基づく場合
3. 人の生命・身体・財産の保護に必要で、本人の同意を得ることが困難な場合
4. 事業承継に伴って情報が移転される場合

本サービス内のECリンクにはアフィリエイト情報が含まれる場合があります。リンクを開くと、リンク先の事業者がその事業者の方針に基づいてアクセス情報を取得します。

## 10. 安全管理措置

当方は、情報の漏えい・滅失・毀損を防止するため、次の措置を講じます。

- HTTPSによる通信
- Firebase Security Rulesと認証によるアクセス制御
- APIキーをSecret Manager等で管理
- クライアントへ秘密鍵を埋め込まない設計
- 外部API呼び出しのCloud Functionsへの集約
- 入力値の検証と利用回数制限
- 公開価格情報からのユーザーID分離
- 保存期間を制限するTTL

インターネット上の通信・保存について、完全な安全性を保証するものではありません。

## 11. ユーザーによる選択と権利

ユーザーは、次の操作を行えます。

- カメラ、位置情報、通知の権限を端末設定から変更する
- 匿名の利用状況分析をアプリ設定から停止する
- アカウントをメールアドレスへ連携するか選択する
- アプリ設定からアカウントと関連データの削除を開始する
- 自己に関する情報の開示、訂正、利用停止、削除等を問い合わせる

削除方法は[アカウントとデータの削除](data-deletion.html)を確認してください。

匿名アカウントをメールアドレスへ連携していない場合、問い合わせだけでは対象アカウントを特定できないことがあります。

## 12. 未成年者

本サービスは13歳以上の方を対象とします。未成年者は、必要に応じて保護者など法定代理人の同意を得て利用してください。

## 13. 本ウェブサイト

本ウェブサイトはGitHub Pagesで配信します。当方は、本ウェブサイト上でCookie、広告、独自のアクセス解析、問い合わせフォームを使用しません。

メールリンクから問い合わせた場合、メールアドレスと本文は利用者が使用するメールサービスおよび当方のメールサービスで処理されます。

## 14. 漏えい等が発生した場合

情報の漏えい・滅失・毀損等が発生した場合、当方は影響範囲の確認、被害拡大防止、関係機関への報告、ユーザーへの周知など、法令と状況に応じた対応を行います。

匿名アカウントでは個別に連絡できない場合があるため、本ウェブサイト、アプリ内表示、ストアのリリース情報等で案内することがあります。
